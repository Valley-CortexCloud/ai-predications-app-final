# Portfolio Data Directory

This directory contains the live portfolio state and proposed changes from the Portfolio Intelligence Engine.

## Files

### `tracker.csv` - Current Portfolio State
Template columns:
- `symbol`: Stock ticker symbol
- `entry_date`: Date position was opened (YYYY-MM-DD)
- `entry_price`: Price at entry
- `shares`: Number of shares held
- `current_price`: Latest price (updated by portfolio_tracker.py)
- `pnl_pct`: Profit/loss percentage
- `days_held`: Days since entry
- `last_exit_score`: Most recent Grok exit score (0-100)

### `proposed_YYYY-MM-DD.csv` - Weekly Validation Output
Generated by `portfolio_validator.py`, contains:
- `symbol`: Stock ticker
- `action`: BUY, SELL, HOLD, or WATCH
- `exit_score`: Grok exit score (0-100)
- `days_held`: Days position has been held
- `reason`: Grok's rationale for the recommendation
- `replacement`: Suggested replacement symbol (if action=SELL)

### `confirmed_YYYY-MM-DD.csv` - Human-Approved Changes
Created manually by reviewing `proposed_*.csv` and approving/modifying actions.
Same format as proposed file.

### `orders_YYYY-MM-DD.json` - Trade Orders
Generated by `trade_executor.py` for submission to Alpaca.

## Workflow

1. **Monday Morning**: `portfolio_validator.py` runs automatically via GitHub Actions
2. **Human Review**: Review `proposed_*.csv` and save as `confirmed_*.csv`
3. **Execute Trades**: Run `trade_executor.py` with confirmed file
4. **Update Tracker**: `portfolio_tracker.py` syncs fills and updates state

## Usage

```bash
# Initialize empty portfolio
python scripts/portfolio_tracker.py --init

# Sync current prices and P&L
python scripts/portfolio_tracker.py --sync

# Generate performance report
python scripts/portfolio_tracker.py --report
```
